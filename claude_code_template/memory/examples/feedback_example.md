## 커밋 관련
- 코드 변경 후 자동으로 커밋하지 말 것 — 사용자가 명시적으로 요청할 때만
- 커밋 메시지는 한국어로 작성 (타입은 영어: Feat, Fix, Refac 등)

## 코드 스타일
- 기존 코드에 불필요한 docstring, 타입 힌트 추가하지 말 것
- 요청한 변경만 수행 — 주변 코드 "개선" 금지
- import 정렬은 ruff에 맡기기 (수동으로 정리하지 말 것)

## 설명 스타일
- 완료 후 작업 요약 불필요 (diff가 설명을 대신함)
- "혹시 다른..." 같은 마무리 멘트 불필요
- 에러 발생 시 원인 + 해결 방법만 간결하게

## 도구 사용
- 파일 읽을 때 Read 사용 (cat 금지)
- 검색은 Grep/Glob 사용 (bash grep/find 금지)
- 대규모 탐색은 Explore 에이전트 활용
