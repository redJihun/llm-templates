# 컨텍스트 윈도우 최적화 규칙

<!--
  Claude Code의 컨텍스트 윈도우(200k)를 효과적으로 활용하기 위한 규칙.
  모든 컨텍스트 파일(CLAUDE.md + rules + memory)은 매 턴 로드됨.

## 컨텍스트 예산 가이드

| 영역 | 권장 비율 | 토큰 수 (200k 기준) |
|------|----------|-------------------|
| System prompt + tools | ~10% | ~20k (자동, 제어 불가) |
| CLAUDE.md + rules + memory | ~5-8% | ~10-16k |
| Autocompact buffer | ~15% | ~30k |
| 작업 가용 공간 | ~67-70% | ~134-140k |

## 파일 크기 가이드

- **CLAUDE.md**: 3,000-5,000 토큰 (핵심만)
- **rules 파일 합계**: 2,000-4,000 토큰
- **memory 파일 합계**: 2,000-5,000 토큰
- **개별 rule 파일**: 300-600 토큰

## 토큰 절약 기법

### CLAUDE.md에 넣지 말 것
- 코드에서 유추 가능한 정보 (함수 시그니처, 파일 경로 등)
- Git 히스토리에서 볼 수 있는 정보
- 전체 디렉토리 트리 (핵심 구조만)
- 모든 API 엔드포인트 상세 스펙 → docs/ 참조 링크로 대체

### 문서 참조 패턴
```markdown
<!-- 나쁜 예: CLAUDE.md에 전체 내용 인라인 -->
## API 상세 스펙
POST /api/v1/users
  Request: { "name": "string", "email": "string", ... }
  Response: { "id": "string", "name": "string", ... }
  ...

<!-- 좋은 예: 링크만 제공 -->
## API 문서
- [API 스펙](docs/api-spec.md) — 전체 엔드포인트 상세
```

## 세션 관리 명령

| 명령 | 시점 | 효과 |
|------|------|------|
| `/compact` | 긴 작업 중간 | 이전 대화 압축, 핵심만 유지 |
| `/clear` | 작업 전환 시 | 전체 초기화, 새 시작 |
| `/context` | 수시 확인 | 현재 컨텍스트 사용량 시각화 |
| `/cost` | 비용 확인 | 현재 세션 토큰/비용 조회 |

## 서브에이전트 컨텍스트 격리

서브에이전트에게 최소 필요 정보만 전달:

```
# 나쁜 예 — 모든 컨텍스트를 에이전트에 전달
"프로젝트 전체 구조를 파악하고 CLAUDE.md를 읽고 모든 rules를 확인한 후..."

# 좋은 예 — 필요한 파일만 지정
"issuance_be_fastapi/routers/auth.py 파일의 login 엔드포인트에서
JWT 토큰 생성 로직을 확인해줘"
```

## 대용량 파일 처리

- 500줄 이상 파일: `Read`의 `offset`/`limit` 파라미터로 부분 읽기
- 로그 파일: `tail` 또는 `Grep`으로 관련 부분만 추출
- 대량 검색 결과: `head_limit`으로 상위 N개만 확인
